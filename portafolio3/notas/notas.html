//INTRUCCION DE TODO LO QUE ESTA EN CADA VUE //API REST: //La Transferencia de Estado Representacional (REST) es un estilo arquitectónico utilizado para compilar servicios web. //En REST, las solicitudes se realizan a través de HTTPS utilizando los verbos HTTP estándar, como GET, POST, PUT, PATCH y DELETE. //Estos verbos se utilizan para realizar operaciones en los recursos de una aplicación. //Una API REST se define mediante los siguientes elementos: //Identificador URI base: Es la URL base que identifica la ubicación de la API. //Métodos HTTP: Estos métodos, como GET, POST, PUT, PATCH y DELETE, se utilizan para realizar acciones sobre los recursos de la API. //Tipo de medio para los datos: Se utiliza un formato de datos, como notación de objetos JavaScript (JSON), //para representar la información enviada y recibida a través de la API. //El uso de una API REST permite una comunicación eficiente entre el cliente y el servidor, siguiendo los principios de estado sin sesión y comunicación sin estado. //Esto hace que las API REST sean ampliamente utilizadas en la industria para desarrollar aplicaciones web y móviles. //Autenticación de API: //La autenticación de API es una capa de seguridad utilizada para permitir a los usuarios iniciar sesión, proteger la privacidad y controlar los permisos de los usuarios en una aplicación. //La implementación de la autenticación es esencial para proteger información confidencial y garantizar que solo los usuarios autorizados tengan acceso a los recursos protegidos. //La Autenticación Basada en Token es un enfoque comúnmente utilizado en la autenticación de API. En este método, //se intercambian tokens entre el cliente y el servidor para validar la identidad del usuario y permitir el acceso a los recursos protegidos. //Los tokens son cifrados y firmados para prevenir la manipulación o suplantación de identidad. //Es importante implementar la autenticación de manera segura, utilizando algoritmos de cifrado y firma robustos, y evitando almacenar tokens en lugares vulnerables. //Además, la gestión adecuada de contraseñas y la protección contra ataques también son factores clave a considerar en la seguridad de la autenticación de API.
